# Raw Sources

This folder stores original engineering source material exactly as received.

## What belongs here

- Standards, handbook chapters, supplier documents, qualification reports, and design memos in their original form
- Scan files, PDFs, spreadsheets, and source packages with provenance metadata
- Intake manifests that identify revision, owner, and acquisition date

## What should not go here

- OCR output or cleaned markdown
- Distilled rules, summaries, or agent interpretations
- Edited copies that no longer match the received source

## How it fits into the AVA pipeline

`raw/` is the provenance anchor for the full system. Every downstream extract, concept, and rule should be traceable back to one or more items in this folder.
